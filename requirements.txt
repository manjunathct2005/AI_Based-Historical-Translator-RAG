# ============================================================================
# requirements.txt  (Streamlit Community Cloud safe profile)
#
# Fixes applied after real deployment failures on Streamlit Cloud, which now
# provisions Python 3.14 and installs with `uv` (not plain pip):
#
#  1. Removed `--extra-index-url .../whl/cpu` for torch: uv's resolver
#     treats extra indexes very differently from pip and was picking
#     incompatible/nonexistent versions of unrelated pinned packages
#     (e.g. requests==2.32.3) from that index, failing resolution entirely.
#     Trade-off: torch now installs from plain PyPI (larger download, but
#     it actually resolves).
#  2. Dropped `easyocr`: it drags in `torchvision` (a second large package
#     that must match your exact torch build) on top of an already heavy
#     stack, and was a major contributor to slow/failed installs. Tesseract
#     (via pytesseract, a tiny pure-Python binding) is now the default and
#     only bundled OCR engine for cloud deploys. EasyOCR still works in the
#     code (utils/ocr.py) if you `pip install easyocr torchvision` yourself
#     -- e.g. for a Docker/local/HF Spaces deploy with more resources.
#  3. Dropped the `[html_clean]` extra on lxml: it was forcing a source
#     build (no prebuilt wheel for bleeding-edge Python versions) which
#     needs libxml2/libxslt *development* headers that plain `apt install
#     libxml2/libxslt1` doesn't provide. trafilatura/readability-lxml don't
#     need that extra, so it's simply removed rather than chasing dev
#     headers for a Python version (3.14) most ML packages don't support
#     yet anyway.
#  4/5. IMPORTANT: torch/transformers/sentence-transformers/faiss do not
#     yet reliably ship wheels for Python 3.14. `runtime.txt` in this repo
#     asks for 3.11, but some Streamlit Cloud accounts only honor that via
#     the "Python version" dropdown in the app's Advanced settings at
#     deploy time, not the file. **You must set Python to 3.11 there** (see
#     README "Deploying to Streamlit Community Cloud") or these packages
#     will keep failing regardless of what's pinned below.
# ============================================================================
